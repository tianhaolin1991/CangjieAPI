ohos.web.webview (Webview)

提供web控制能力，组件提供网页显示的能力。

需要权限

访问在线网页时需添加网络权限：ohos.permission.INTERNET。

导入模块

import ohos.webview.*

class WebviewController

public class WebviewController {
    public init()
    public init(webTag: String)
}

通过WebviewController可以控制Web组件各种行为。一个WebviewController对象只能控制一个Web组件，且必须在Web组件和WebviewController绑定后，才能调用WebviewController上的方法（静态方法除外）。

init()

public init()

用于创建WebviewController对象的构造函数。

系统能力： SystemCapability.Web.Webview.Core

init(String)

public init(webTag: String)

用于创建WebviewController对象的构造函数。

系统能力： SystemCapability.Web.Webview.Core

参数：

参数名类型必填说明
webTagString是指定了Web组件的名称，默认为Empty。

func loadUrl(String)

public func loadUrl(url: String): Unit

加载指定的URL。

系统能力： SystemCapability.Web.Webview.Core

参数：

参数名类型必填说明
urlString是需要加载的URL。

错误码：

以下错误码的详细介绍请参见webview错误码。

错误码ID错误信息
17100001Init error. The WebviewController must be associated with a Web component.
17100002Invalid url.
17100003Invalid resource path or file type.

示例：

package ohos_app_cangjie_entry
import ohos.base.*
import ohos.component.*
import ohos.state_manage.*
import ohos.state_macro_manage.*
import ohos.webview.*

@Entry
@Component
class EntryView {
    let webController = WebviewController()
    func test() {
        Column(10) {
            Button("loadUrl")
            .onClick {
                evt =>
                AppLog.info("loadUrl")
                webController.loadUrl("www.example1.com")
            }.width(400.px).height(150.px)

            Web(src: "www.example.com", controller: webController)
            .onPageBegin({evt =>
                AppLog.info("page begin url: ${evt.url}")
            })
            .onPageEnd({evt =>
                AppLog.info("page end url: ${evt.url}")
            })
        }
    }
}

func registerJavaScriptProxy(Array<(String) -> String>, String, Array<String>)

public func registerJavaScriptProxy(funcs: Array<(String) -> String>, name: String, methodList: Array<String>): Unit

注入Cangjie方法到window对象中，并在window对象中调用该方法。注册后，须调用refresh接口生效。

系统能力： SystemCapability.Web.Webview.Core

参数：

参数名类型必填说明
funcsArray<(String) -> String>是参与注册的应用侧Cangjie方法数组。注册的Cangjie方法的入参和返回值都是String类型。
nameString是注册Cangjie方法数组的名称，与window中调用的对象名一致。注册后window对象可以通过此名字访问应用侧Cangjie方法。
methodListArray<String>是参与注册的应用侧Cangjie方法名，此数组的长度需要与funcs数组一致。注册完成后，后续funcs的判等会通过methodList来判断。因此后续如果想注册新的、或更改funcs，需要传入新的methodList。

错误码：

以下错误码的详细介绍请参见webview错误码。

错误码ID错误信息
401WebviewController registerJavaScriptProxy failed: the funcs and methodList must have the same size.
17100001Init error. The WebviewController must be associated with a Web component.

注意：

  • 所有拥有仓颉函数类型的实体均支持被注册为JavaScript对象的方法，包括：全局函数、局部函数、实例/静态成员函数和闭包。
  • 如果方法名称列表methodList中存在重名的方法名称，例如myMethod，则调用JavaScript对象的myMethod方法时，将按照methodList中的顺序，选择第一个名称为myMethod的方法进行调用。
  • 目前仅支持函数签名为(String) -> String的仓颉函数被注册为JavaScript的方法。
  • 支持多次使用相同的name定义JavaScript对象，每一次调用该接口进行JavaScript对象注册时，如果methodList与上一次注册时使用的methodList相等，则不会重新定义JavaScript对象；否则将覆盖前一次的JavaScript对象定义。
  • 方法名称列表methodList中如果存在空字符串或以数字开头等的不能作为合法JavaScript标识符的字符串，在调用该接口进行JavaScript方法注册时并不会抛出异常，而是在相关JavaScript源码被执行时导致相关问题。
